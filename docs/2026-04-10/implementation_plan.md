# 釣果画像チェンジャー（FishMagazineCoverGenerator） 実装計画書

開発仕様に基づく「ブツ持ち写真（釣果写真）」の「魚」を自動検出し、ブラウザ上でサイズ変更するWebアプリケーション（SPA）の実装計画。サーバーコスト0円を目指し、クライアントサイドAIを活用する。

## 要件の確認と不明点・相談事項
ユーザーの方による要件から、以下の3点について提案および相談があります。これらが合意でき次第、実装へ進みます。

> [!IMPORTANT]
> 1. **フロントエンドフレームワークの選定**: サーバー処理が不要な完全なSPA（Single Page Application）であり、かつGitHub Pages等へのデプロイを想定しているため、Next.jsよりも軽量でクライアントサイド開発に特化した **Vite + React (TypeScript)** の構成を推奨します。こちらで進めてもよろしいでしょうか？
> 2. **AIモデルの選定**: 仕様書にある `segformer` は、一般的な風景や物体（車・人・空など）の切り抜きには向いていますが、「魚」専用のクラスがないため精度が出ない可能性があります。代替として、Transformers.jsで利用可能でテキストから画像を切り抜けるゼロショットセグメンテーションモデル **`Xenova/clipseg-rd64-refined`** の使用を提案します。「fish」というテキストを与えて魚だけを切り抜くことが可能です。こちらを採用してもよろしいでしょうか？
> 3. **修復処理（Inpainting）のロジック**: 「本格的なAI修復は行わない」とのことですが、魚を縮小した時に元の大きな魚が背景に残らないようにする必要があります。Canvas APIを用いて「切り抜いた穴の周囲のピクセルを内側に引き伸ばす/ぼかして埋める」といった簡易的なインペイント処理を独自実装する方針でよろしいでしょうか？

## アーキテクチャと技術スタック

- **開発言語**: TypeScript / HTML / CSS
- **フレームワーク**: Vite + React
- **スタイリング**: Vanilla CSS または TailwindCSS (デフォルトではモダンに仕上げやすいTailwindを提案しますが、プレーンなCSSの要望があれば合わせます)
- **AI推論**: `@xenova/transformers` (Transformers.js)
  - 対象タスク: Image Segmentation (Zero-shot)
  - 利用モデル(案): `Xenova/clipseg-rd64-refined`
- **画像加工**: HTML5 Canvas API

## 構成コンポーネント

### UIレイヤー
以下の機能をReactコンポーネントとして実装します。
- **ImageUploader**: ドラッグ＆ドロップまたはファイル選択ダイアログによる画像読み込み。
- **Workspace**: Canvasを表示し、画像のプレビューと加工状況を可視化する。
- **Controls**: 「AIで魚を検出する」ボタン、変形量調整のスライダー（例: 0.5倍〜2.0倍）、および「画像を保存」ボタン。

### ロジックレイヤー
- **AI Segmenter Service**: `pipeline` を初期化し、画像をテンソルに変換してセグメンテーションを実行、マスク配列（白黒画像データ）を返す。（メインスレッドをブロックしないようWeb Worker上での実行を想定）
- **Image Editor Engine**: Canvas上で以下の処理を行う。
  - AIの生成したマスクに基づいて魚部分の切り抜き。
  - 元画像の魚があった位置の簡易的な背景塗りつぶし（修復設定）。
  - 切り抜いた魚画像のスケーリング（拡大・縮小）と再配置。

## 開発フェーズ（タスク定義）
開発は以下のステップで行います。段階的な検証を挟むことで手戻りを防止します。

1. プロジェクトの初期セットアップ (Vite + React)
2. UIのモックアップ作成 (Canvasプレビュー画面と操作パネル)
3. Transformers.js の導入とセグメンテーション処理の検証
4. Canvasによる画像切り抜きロジックの実装
5. Canvasによる画像インペイント（簡易修復）と合成・保存ロジックの実装
6. 最終調整（パフォーマンス・レスポンシブデザイン）
